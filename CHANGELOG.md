## Versioning

This repository follows Semantic Versioning:

- **MAJOR** version when incompatible documentation structure changes are introduced.
- **MINOR** version when new documentation is added in a backward-compatible manner.
- **PATCH** version for corrections, clarifications, formatting improvements, or non-breaking updates.
